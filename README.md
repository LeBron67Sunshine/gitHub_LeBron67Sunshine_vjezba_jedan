# gitHub_LeBron67Sunshine_vjezba_jedan

## 1. Što je Visual Studio Code i za što se najčešće koristi?
To je besplatan i lagan program za uređivanje kodova koji je razio Microsoft i on se najčešće koristi za programiranje

## 2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
Explorer

## 3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?
U lijevom izborniku odaberemo 5. ikonu "Extensions" i kada odaberemo neku ekstenziju kliknemo gumb install; one nam davaju puno više mogućnosti tijekom pisanja koda

## 4. Kako otvoriti postojeći projekt ili mapu u VS Code-u?
File -> Open file

## 5. Što se sve može raditi u Exploreru?
Možemo gledati strukturu datoteka, kopirati ih i brisati

## 6. Kako možeš brzo pronaći određenu datoteku u projektu?
Pretragom tako što pritisnemo CTRL + P i unesemo ime tražene datoteke

## 7. Kako možeš istovremeno gledati dvije različite datoteke u editoru?
Pritisnemo CTRL + SHIFT + P i odaberemo split editor

## 8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?
Pritisnemo CTRL + ALT + F

## 9. Što je version control i zašto se koristi?
Koristi se za praćenje promjena i omogućuje ulazak u prethodne verzije koda

## 10. Kako se u VS Code-u otvara panel za Git (Source Control)?
U lijevom izborniku pritisnemo 3. ikonu grane

## 11. Što znače commit, push, pull?
Commit – sprema promjene lokalno
Push – šalje lokalne commitove na GitHub
Pull – preuzima promjene s GitHuba u lokalni projekt

## 12. Čemu služi commit poruka i kako treba izgledati?
Commit poruka opisuje što si promijenio i ona treba biti kratka, jasna i konkretna

## 13. Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?
Promjene vidiš u Source Control panelu ili klikom na datoteku pa gumb Open Changes

## 14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
1. Kreiraš datoteku
2. U Source Control označiš promjene (stage)
3. Napraviš commit s porukom
4. Push da pošalješ promjene na GitHub

## 15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
Na GitHubu odeš u datoteku, onda odeš na History, odabereš commit pa onda odabereš Restore

## 16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
Status projekta provjeravaš naredbom git status u terminalu ili pregledom izmjena u Source Control panelu
